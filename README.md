# Love Nikki Ranking Site

Story and Commission (guild) ranking leaderboards for Love Nikki, served as a
static site on GitHub Pages.

The ranking data is generated by a separate pipeline:
[love-nikki-ranking-extractor](https://github.com/syngtu/love-nikki-ranking-extractor).

## Stack

React + TypeScript + Vite. The generated leaderboard JSON lives in
`public/data/` (committed) and is produced by the extractor.

## Development

```bash
npm install
npm run dev      # local dev
npm run build    # production build -> dist/
```

## Deployment (GitHub Pages)

Pushing to `main` triggers `.github/workflows/deploy.yml`, which builds the app
and publishes `dist/` to GitHub Pages.

The site is served from a project subpath
(`https://<user>.github.io/love-nikki-ranking-site/`), so Vite's `base` is set
to `/love-nikki-ranking-site/` in `vite.config.ts`. If you deploy to a user/org
page or a custom domain, build with `VITE_BASE=/` (or your subpath).

One-time repo setup: in **Settings → Pages**, set **Source** to **GitHub Actions**.

## Updating the data

The leaderboard JSON in `public/data/` is produced by the
[love-nikki-ranking-extractor](https://github.com/syngtu/love-nikki-ranking-extractor)
repo. Check it out as a sibling of this repo:

```
personal/
├── love-nikki-ranking-extractor/   # data pipeline
└── love-nikki-ranking-site/        # this repo (GitHub Pages)
```

Run the extractor's pipeline, which writes directly into this repo's
`public/data/`, then commit and push the regenerated JSON here to publish the
update. See the extractor repo's README for details.
